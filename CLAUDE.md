## High‑Level Architecture

- **Entry point**: `src/app/page.tsx` – a client‑side component that renders the main UI. It contains a search input that triggers a call to the `/api/latitude/[city]` route to fetch geographic coordinates.
- **API routes** (under `src/app/api`):
  - `src/app/api/latitude/[city]/route.ts` – receives a city name, forwards the request to the OpenWeatherMap geocoding endpoint, and returns the latitude/longitude.
  - `src/app/api/current/[weather]/route.ts` – expects `lat` and `lon` query parameters, calls the OpenWeatherMap One Call API (`process.env.API_KEY` must be set) and returns the weather JSON.
- **Layout**: `src/app/layout.tsx` (standard Next.js layout, not modified).
- **Styling**: Tailwind CSS is installed (`tailwindcss` and `@tailwindcss/postcss`), but the project does not contain a `tailwind.config.js` file – Tailwind’s default configuration is used.
- **Dependencies**: Next.js 16, React 19, TypeScript 5. No additional runtime libraries.

### Data Flow Overview
1. User types a city → `handleChange` in `page.tsx` calls `/api/latitude/:city`.
2. API returns `{ lat, lon }` → stored in `geoData` state.
3. `useEffect` watches `geoData`; when available it calls `/current?lat=&lon=` to fetch weather data.
4. Weather data is logged to the console (currently not displayed in the UI).
